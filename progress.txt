# Progress Log

## Codebase Patterns
- `NAV_ITEM_CONFIG` in web_server.py (line ~235) defines all top-nav items as (key, href, label, tone_css_class) tuples
- `_build_nav_items()` in web_server.py constructs nav items with active state; `_build_page_context()` assembles full template context
- `base_research.html` uses Jinja2 template inheritance with blocks: head, hero, content, scripts
- CSS tone classes (`.tone-<key>.active/.tone-<key>:hover`) in `static/research_ui.css` provide per-nav-item accent colors
- Navigation is data-driven: change `NAV_ITEM_CONFIG` and all pages that extend `base_research.html` get the updated nav
- Old pages (Track, Explore, Insights) still have Flask routes but no longer appear in top nav
